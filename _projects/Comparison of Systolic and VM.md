## Comparison between Systolic Array and Vector Multiplier based on FPGA Implementation üõ†Ô∏è

This project pits **systolic arrays** against **vector multipliers** for matrix multiplication on Xilinx's `xcu250` FPGA. Let‚Äôs break it down! üß†

---

### <br>Introduction üéØ
With AI accelerators booming, efficient *matrix multiplication units (MXUs)* are critical. We compare:
- **Systolic Arrays**: Inspired by Google‚Äôs TPUv1
- **Vector Multipliers**: GPU-style adder-tree designs

Tested on matrix sizes from `8√ó8` to `64√ó64`, we dive into *timing*, *resources*, and *scalability*. üìà

---

### <br>Architectural Highlights üèóÔ∏è
- **Systolic Array**: A grid of PEs with *built-in pipelining*. Weights stay put, data flows‚Äîsimple yet powerful! üîÑ
- **Vector Multiplier**: Direct vector ops with an adder tree. Needs *explicit pipelining* for big matrices. üßÆ

---

### <br>Implementation Snapshot üîß
- **FPGA**: `xcu250` UltraScale+
- **Matrix Sizes**: `8√ó8`, `16√ó16`, `32√ó32`, `64√ó64`
- **Vivado Flags**: `dont_touch`, `use_dsp`, `ram_style=block`

---

### <br>Results Roundup üìä
#### Resource Use
- **Systolic Array**: Steady rise‚Äîe.g., `4096 DSPs` at 64√ó64.
- **Vector Multiplier (Non-pipelined)**: Leaner but *timing flops* at scale.
- **Vector Multiplier (Pipelined)**: Timing improves, but *LUTs soar* (e.g., `36166` at 64√ó64).

### <br>Key Takeaways üîç
- **Scalability**: Systolic arrays win with *consistent timing* and efficient growth. üåü
- **Trade-offs**: Vector multipliers need *heavy pipelining*, spiking resource use for big tasks.

---

### <br>Conclusion üèÅ
- **Systolic Arrays**: Champs for *large-scale matrix ops*. üí™
- **Vector Multipliers**: Fit for *smaller, parallel setups* (think GPUs).

> *In a nutshell: Systolic arrays rule for big matrix crunching; vector multipliers shine in compact roles!* üöÄ
